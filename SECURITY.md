# Security Policy

## Supported Versions

Currently this component is in alpha stage.

Unfortunately [`shellcheck`](https://www.shellcheck.net/) does not support `zsh`, so the code not automatically checked against known anti-patterns and potential weaknesses.

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you however want to report a vulnerability, please use the GitHub issue tracker.

Or the private vulnerability reporting form provided by GitHub

1. On GitHub, navigate to the main page of the repository.
1. Under the repository name, click Security.
1. Click Report a vulnerability to open the advisory form.
1. Fill out the form, providing as much detail as possible.
1. At the bottom of the form, click Submit report. GitHub will display a message letting you know that the maintainers have been notified

See [the documentation](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/report-a-vulnerability/privately-reporting-a-security-vulnerability#privately-reporting-a-security-vulnerability) for more details.
